Getting Started Persona & Scenario Dialogue Structure W++ Formatting Advanced Configuration Testing & Debugging Common Issues Best Practices

1. Getting Started

Before creating your roleplay bot, make sure you have the following prerequisites:

  • LLaMA 3 8B or Nephra Yodayos 8B API Access - Ensure you have access to the required model API.
  • Persona and Scenario Data - Prepare the character and scenario data.
  • Server Setup - Host your bot on a capable server.

Follow the installation instructions to get started.

2. Creating Persona and Scenario Data

Define the core attributes for your persona and scenario. Example:

{ "persona": { "name": "Raven", "age": "25", "appearance": "Long black hair, dark leather outfit", "personality": ["Mysterious", "Cunning", "Sarcastic"], "tone": "Dark and witty" }, "scenario": { "title": "The Shadow Realm", "setting": "A dark, enchanted forest", "goal": "Survive the night and uncover the truth about the forest's curse", "conflict": "Navigate through illusions and traps set by a malevolent spirit." } }

3. Dialogue Structure

The dialogue should guide user interactions and include branching choices.

{ "user_input": "What should I do next?", "bot_response": "The trees begin to move, casting shadows in every direction. 'You have two choices,' Raven whispers. 'You can follow the path to the left or venture into the cave ahead.'", "choices": [ {"text": "Take the path to the left", "next_scenario": "forest_path"}, {"text": "Enter the cave", "next_scenario": "mysterious_cave"} ] }

4. W++ Formatting for Roleplay

W++ formatting ensures the bot responds consistently with its persona and scenario. Here's an example:

{ "persona": { "name": "Lucius the Wizard", "tone": "Wise and gentle", "behavior": "Guides the user with calm, thoughtful advice." }, "scenario": { "title": "The Temple of Knowledge", "setting": "Ancient temple with forgotten scrolls", "goal": "Help the user retrieve a lost artifact" }, "dialogue": [ { "user_input": "How do I open the hidden door?", "bot_response": "Lucius gazes at the stone door, then turns to you with a knowing smile. 'The key is hidden in the stars above. Focus on their alignment, and the path will reveal itself.'" } ] }

5. Advanced Configuration

For more dynamic bots, consider advanced features like:

  • User Memory - Store decisions made by the user for a personalized experience.
  • Dynamic Choices - Generate choices based on the scenario or user input.
  • Response Variation - Avoid repetitive dialogue by introducing variety.

6. Testing and Debugging

Test your bot with various inputs to ensure responses align with the persona and scenario.

  • Test different user inputs.
  • Check for response consistency.
  • Verify that dynamic choices are generated correctly.

7. Common Issues and Troubleshooting

  • Bot Doesn't Follow Persona: Double-check persona settings and tone.
  • Dialogue Feels Repetitive: Introduce response variation and conditional logic.
  • Bot Breaks Character: Reinforce the persona in prompts.
